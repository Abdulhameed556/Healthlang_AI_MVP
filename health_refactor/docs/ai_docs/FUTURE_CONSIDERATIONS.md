# Future Considerations — AI Service

## KB Document Quality & Optimisation

### Continuous Re-evaluation
Every time a knowledge base is updated (new entry indexed or existing entry replaced),
auto-trigger a retrieval evaluation run in the background. If aggregate scores drop below
a configurable threshold, alert the admin via webhook or in-app notification before the
change propagates to live agents.

Deferred because it requires:
- Per-KB threshold configuration stored in the database
- Background job scheduler (beyond the current Dramatiq ingest worker)
- Notification delivery system (email / webhook / in-app)

### AI-Assisted Document Restructuring (MVP implemented as `/suggestions`)
The current `POST /retrieval-evaluation/{run_id}/suggestions` endpoint surfaces
low-relevancy chunks and asks an LLM to recommend how to split the original document
into focused sub-files. The admin can then:

1. Review highlighted noisy chunks in the demo UI
2. Accept the suggested split (e.g. `Architecture.docx`, `Pricing.docx`)
3. Re-upload the smaller, focused files and re-index them
4. Re-run evaluation to confirm scores improved

**Next iteration ideas:**
- Let the admin download a pre-split version of the document generated by the LLM
- Auto-apply the split and trigger re-indexing with a single click
- Show a before/after score comparison after re-upload
